# Lab Environment

## Purpose
This lab environment was used to simulate authentication abuse, brute-force activity, username enumeration, and suspicious access patterns for QRadar offense analysis.

## Systems Observed
- KALI (attack simulation host)
- 192.168.101.128
- 192.168.101.139
- 192.168.101.10
- 192.168.101.133

## Technologies Involved
- IBM QRadar SIEM
- Windows authentication logs
- Linux authentication logs
- Honeypot / tarpit alerts

## Simulated Attack Behaviors
- Brute force login attempts
- Password spraying
- Invalid username enumeration
- Root account targeting
- Failed logins followed by success
- Honeypot interaction

## Note
This environment was used for controlled analysis and offense documentation, not production incident reporting.
