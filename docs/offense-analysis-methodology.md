# Offense Analysis Methodology

## Workflow Used

1. Review the offense summary
2. Extract key context (source, destination, usernames, counts)
3. Pivot into related evidence views
4. Identify whether the pattern reflects noise or suspicious behavior
5. Assess operational risk
6. Document findings and next steps

## What Good Offense Analysis Should Answer
For each offense, the analyst should be able to answer:

- What happened?
- Why did QRadar trigger this offense?
- What evidence supports the assessment?
- Is this likely benign, suspicious, or urgent?
- What should be validated next?
