# Observed QRadar Correlation Rules

## Authentication Related Rules
- Multiple Login Failures for the Same User
- Repeated Windows Login Failures
- Repeated Non-Windows Login Failures
- Login Failures Followed By Success
- Root Login Failed
- Bad Username
- Invalid User
- Authentication Failure

## Network / Reconnaissance Related Rules
- Access to Honeypot or Tarpit Defined Address
- New Host Discovered

## Rule Analysis
These rules collectively indicate:
- account targeting
- brute-force activity
- credential abuse
- suspicious access attempts
- network reconnaissance

## Analyst Observation
Many of the triggered rules appear to represent different perspectives of the same broader attack sequence rather than completely independent incidents.
